 当前项目结构

   orchestrator
   ├── FundamentalAgent  → get_fundamentals (财报 + LLM总结)
   ├── MarketAgent       → get_market_data  (行情 + 资金流)
   └── RiskAgent         → get_market_data + get_fundamentals + get_stock_news_summary

  数据层：Tushare / AkShare / Baostock / 
  EastMoney（collector已建，tools层只用了前两个）。状态层（AlphaBeeState）、记忆层（memory/
  ）、工作流层（workflow/）、应用层（apps/）均为空目录。

  -------------------------------------------------------------------------------------------------------

  持续改进方向

  一、回答质量

  1. 工具深度不足

  目前每个 Agent 只有 1-2 个工具，调用一次就完事。投资分析需要多轮、多角度的数据组合：

  ┌───────────────────────────────────┬─────────────────────────────────┬────────────────────────────────┐
  │ 缺失工具                          │ 用途                            │ 数据源                         │
  ├───────────────────────────────────┼─────────────────────────────────┼────────────────────────────────┤
  │ get_historical_prices(symbol,     │ 趋势判断、支撑阻力、均线系统    │ Tushare pro_bar                │
  │ period)                           │                                 │                                │
  ├───────────────────────────────────┼─────────────────────────────────┼────────────────────────────────┤
  │ get_analyst_reports(symbol)       │ 券商一致预期、目标价            │ EastMoney（已有collector）     │
  ├───────────────────────────────────┼─────────────────────────────────┼────────────────────────────────┤
  │ get_peer_comparison(symbol)       │ 同行估值对比（PEG、PS横向比较） │ Tushare daily_basic            │
  ├───────────────────────────────────┼─────────────────────────────────┼────────────────────────────────┤
  │ get_major_holders(symbol)         │ 机构持仓变化、大股东增减持      │ Tushare top_inst/              │
  │                                   │                                 │ stk_holdertrade                │
  ├───────────────────────────────────┼─────────────────────────────────┼────────────────────────────────┤
  │ get_industry_data(industry)       │ 行业景气度、板块轮动            │ Tushare ths_daily/dc_daily     │
  ├───────────────────────────────────┼─────────────────────────────────┼────────────────────────────────┤
  │ search_announcements(symbol)      │ 定增、回购、业绩预告等公告      │ Tushare anns_d                 │
  └───────────────────────────────────┴─────────────────────────────────┴────────────────────────────────┘

  2. Agent Prompt 过于简单

  当前 prompt 只有职责列表，缺少推理框架。好的投资分析 prompt 应该包含：

   - 分析链条："先看行业景气 → 再看公司地位 → 再看财务质量 → 最后看估值"
   - 量化锚点：ROE > 15% 认为优秀，负债率 > 70% 需警惕
   - 输出模板：强制要求给出"结论 + 支撑证据 + 反驳证据 + 不确定性"的结构

  3. 缺乏多周期视角

  当前基本面只取最新一期报告，无法判断趋势。需要拉取 4-8 个季度数据，分析：

   - 利润是否持续增长还是单季爆发
   - 现金流与利润是否匹配（利润质量）
   - 负债率是否在上升

  4. 缺乏交叉验证

  三个子 Agent 各自独立输出，Orchestrator 只是简单汇总。缺少：

   - 基本面与行情数据的一致性校验（财报好但股价连跌 → 需解释）
   - 新闻与量化数据的事件驱动关联

  -------------------------------------------------------------------------------------------------------

  二、系统能力

  5. Memory 层完全为空

  memory/ 目录为空，但 mem0ai 已在依赖里。可以实现：

   - 用户偏好记忆：风险偏好、关注行业、持仓情况
   - 对话历史：记住上下文，支持追问（"这只股票上次分析后又有什么变化？"）
   - 分析结论缓存：同一只股票 1 小时内不重复拉数据

  6. Workflow 层为空

  workflow/ 有目录结构但无实现。当前是 single-shot 问答，应该升级为：

   - 定时监控工作流：每日盘后自动分析持仓，生成风险预警
   - 事件驱动工作流：新闻触发 → 自动判断是否影响持仓
   - 批量研究工作流：扫描全市场满足特定条件的股票

  7. 缺乏不确定性表达

  LLM 倾向于给出确定性结论，投资场景必须强制表达置信度：

   - "数据截至 XX，最新公告可能已有变化"
   - "此分析基于历史财务数据，不构成投资建议"
   - 对于数据缺失字段（northbound_flow=0、concepts=[]）应在回答中标注

  8. 技术分析完全缺失

  Technical 模型已定义但注释掉了（# technical: Technical）。MA5/MA10/MA20 
  数据、MACD、RSI、布林带等技术指标对短期交易判断很关键，目前 MarketAgent
  拿不到任何趋势信息。

  -------------------------------------------------------------------------------------------------------

  三、工程质量

  9. 错误处理太弱

  当前任何 Tushare API 失败都会让整个请求崩溃。应该：

   - 每个工具单独 try/except，返回部分数据 + 错误说明
   - Agent 感知到数据缺失后，在回答中说明局限性而不是报错

  10. MCP Server 为空

  mcp_server/ 目录是空的。实现 MCP 后可以把这些工具暴露给 Claude、Cursor 
  等客户端直接调用，大幅扩展使用场景。

  -------------------------------------------------------------------------------------------------------

  优先级建议

   短期（提升回答质量）:
     → 补充历史多期财务数据（4-8个季度趋势）
     → 加 get_analyst_reports + get_peer_comparison 工具
     → 重写三个 Agent 的 prompt（加分析框架和量化锚点）
     → 让 Orchestrator 做交叉校验而不是简单汇总
   
   中期（系统能力）:
     → 接入 Memory（用户画像 + 对话上下文）
     → 技术分析工具（均线、MACD、量能）
     → 错误降级处理（部分数据缺失时仍能给出有限分析）
   
   长期（产品化）:
     → Workflow 定时监控 + 事件驱动
     → MCP Server 暴露工具
     → Apps 层前端界面
